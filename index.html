Welcome to Groovy WebLogic Scripting Tool

GWLST is the Groovy cousin of Oracle WebLogic Scripting Tool (WLST), used to configure and manage WebLogic Server domains, servers and services.

WLST, JMX and MBeans

WLST is really two partially overlapping tools with a joint interface:

  • In "offline mode", it manipulates a domain directory directly on a local file system.
  • In "online mode", it connects to a running WebLogic Server over JMX.

WebLogic Server exposes its configuration and management interfaces as WebLogic Server MBeans that are accessible remotely over JMX. The WebLogic Console exposes a subset of this interface; a JMX client can access anything that can be accessed through WebLogic Console and more.

The great irony of Java Management Extensions (JMX) is that the Java programming language is poorly suited for writing a JMX client application because it lacks the dynamic language features to hide the fact that the remote MBean is not an ordinary object. Whereas WLST Online scripts are not pretty, they are still often less painful to write than developing custom management utilities with JMX in Java.

GWLST and Groovy

GWLST is a pure JMX client, covering only the "online" aspect of WLST, but it does this more elegantly than either Java or WLST.

Whereas WLST is a module on top if Jython, GWLST is a DSL embedded in the Groovy programming language. Because Groovy compiles to Java Byte Code and runs on the JVM, it integrated seamlessly with other code written in Java.

Groovy has dynamic language features allowing classes to "pretend" having properties and methods. The caller (written in Groovy) can therefore access MBeans as if they were ordinary objects rather than using the verbose reflection style. This is demonstrated in the simple GroovyMBean, but GWLST takes it further by adding navigation (an MBean returns another MBean) and WebLogic metadata.

Summary

  • GWLST can be used as a scripting engine like WLST Online
  • GWLST can be used as a library classes written in Groovy that are compiled and run seamlessly on the JVM
  • GWLST scripts are typically a fraction of the length or either its counterpart written in WLST or Java
  • GWLST is so expressive that it is useful to WebLogic administrators "on the fly" as an alternative to otherwise repetitive actions through the WebLogic Console
